# Editorial policy

## Purpose

This site is a curated guide, not a scrape dump.

## Current policy

- keep the data set small and legible
- prefer accurate, current venue names and links
- avoid duplicating the same event across multiple slugs
- use one canonical slug per venue and event
- write descriptions that are factual and neutral
- link back to the source listing for each event
- show freshness as source-check metadata, not as a marketing claim

## Future policy

When ingestion starts, manual review should remain available for:

- ambiguous venue names
- duplicate events
- cancelled or moved shows
- stale ticket links

The first automatic pass should fail closed rather than publishing uncertain data without review.

The site should not claim complete Sheffield coverage until source breadth and import health justify it.
